## 🎯 功能说明

### AI服务配置

#### 提供商选择
- 下拉选择器展示所有可用提供商
- 显示提供商状态（已配置/未配置）
- 显示提供商友好名称（OpenAI、智谱AI等）
- 支持动态切换默认提供商

#### 功能开关
- **启用AI功能** - 主开关，控制所有AI功能
- **降级到传统方法** - AI失败时自动使用传统方法
- **启用成本限制** - 控制每日成本预算
- **启用AI缓存** - 缓存AI响应提高性能

#### 高级设置
- **每日成本限制** - 设置成本阈值（美元）
- **缓存过期时间** - 设置缓存TTL（秒）

#### 操作按钮
- **测试连接** - 测试当前选择的AI提供商是否可用
- **保存配置** - 保存所有AI设置
- **重置为默认值** - 恢复默认配置

### AI使用统计

#### 统计指标
- **总成本** - 累计AI使用成本（美元）
- **总请求数** - AI请求总数
- **总Token数** - 消耗的Token总数
- **今日成本** - 当日使用的成本

#### 各提供商统计
- 显示每个提供商的使用成本
- 支持多提供商成本对比

## 🚀 使用方法

### 1. 配置AI提供商

1. 进入"系统设置"页面
2. 找到"AI服务配置"卡片
3. 在"默认AI提供商"下拉框中选择提供商
4. 未配置的提供商会显示"未配置"标签

### 2. 配置AI功能开关

- **启用AI功能** - 开启或关闭AI提取和分析
- **降级到传统方法** - AI失败时是否降级
- **启用成本限制** - 设置成本上限
- **启用AI缓存** - 是否缓存AI响应

### 3. 设置成本控制

1. 开启"启用成本限制"开关
2. 设置"每日成本限制"金额（美元）
3. 达到限制时将不再发起AI请求

### 4. 配置缓存设置

1. 开启"启用AI缓存"开关
2. 设置"缓存过期时间"（秒，默认3600）
3. 缓存可以显著提高性能和降低成本

### 5. 测试AI连接

1. 配置好AI提供商后
2. 点击"测试连接"按钮
3. 系统会测试当前提供商是否可用
4. 显示成功或失败消息

### 6. 查看AI使用统计

- 在"AI使用统计"卡片中查看：
  - 总成本、总请求数、总Token数
  - 今日成本
  - 各提供商成本对比
  - 数据自动刷新

## 💾 数据持久化

AI配置数据会保存到：
- 后端数据库（通过API）
- 后端环境变量（.env文件）

前端只负责：
- 提供配置界面
- 发送配置请求
- 显示配置和统计信息

## 🔧 配置文件

### 后端环境变量

在后端 `.env` 文件中配置：

```env
# 默认AI提供商
AI_DEFAULT_PROVIDER=openai

# OpenAI配置
AI_OPENAI_API_KEY=your_api_key
AI_OPENAI_MODEL=gpt-4o-mini

# 智谱AI配置
AI_ZHIPU_API_KEY=your_api_key
AI_ZHIPU_MODEL=glm-4-flash

# AI功能开关
AI_FALLBACK_ENABLED=true
AI_COST_LIMIT_ENABLED=false
AI_COST_LIMIT_DAILY=10.0
AI_CACHE_STRATEGY=memory
AI_CACHE_TTL=3600
```
